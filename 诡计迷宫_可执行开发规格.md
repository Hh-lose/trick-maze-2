# 《诡计迷宫》可执行开发规格 v1.0

## 0. 给 Codex 的执行指令

你是一名独立游戏开发工程师。请在当前工作目录创建一个**可以直接运行的手机端单机 2D 像素 Roguelite 游戏**，游戏名为《诡计迷宫》。严格按本规格实现可玩的首版，而不是只生成原型页面、设计文档或伪代码。

### 强制技术约束

- 使用 **Vite + TypeScript + Phaser 3**。
- 游戏应以浏览器网页运行，手机竖屏优先，适配 360 x 640 至 430 x 932，桌面浏览器也能用键鼠游玩。
- 使用 Canvas 绘制的像素风角色、怪物、地面、墙体和特效。不要依赖外部图片、音频、CDN 或网络服务。
- UI 使用 Phaser 原生文字与图形；全部文案为简体中文。
- 持久化数据使用 `localStorage`，无登录、无联网、无广告、无付费。
- 所有随机数都经过一个可注入种子的随机数工具；每局顶部显示种子，便于复现问题。
- 保持模块化：配置/数据、地图、战斗、实体、奖励、存档、场景、UI 分开。不要把主要逻辑写进单一文件。
- 完成后运行构建检查；修复 TypeScript 和构建错误。启动开发服务器，并在最终说明中给出本地访问地址。

### 必须交付

1. 一个可运行的项目，含 `package.json`、源码和简洁的 `README.md`。
2. 首屏是标题页，可开始新局、继续最高纪录、打开永久装备页、重置存档。
3. 一局内完整实现：房间生成 -> 战斗 -> 掉落/宝箱 -> 技能三选一 -> 下一关 -> Boss -> 死亡结算 -> 存档。
4. 触屏双摇杆、攻击按钮、交互按钮；桌面提供 WASD/方向键、鼠标瞄准、空格攻击、E 交互。
5. 至少能流畅运行到第 10 关。活动怪物上限为 55；不得因关卡数增长让单屏实体无限增加。

## 1. 游戏概念

### 一句话定位

一款中式诡异题材的竖屏像素动作 Roguelite：玩家在不断变形的迷宫房间中清理鬼怪，临时构筑当局能力，并以碎片和金币持续强化下一次闯关。

### 核心体验

- **短循环**：每个普通房间约 60-150 秒，击败敌人、开箱、传送。
- **操作感**：移动躲避、手动朝向攻击、明确的攻击前摇/命中反馈。
- **随机构筑**：武器与技能石只在本局有效，产生不同的战斗方式。
- **长期目标**：金币、装备碎片和神秘宝箱碎片可带出，推动更高纪录。

### 美术与氛围

- 视角：2D 俯视角，像素化块面，16 px 为一个视觉格。
- 主色：墨黑、暗青、纸黄、血红；不要使用明亮渐变或现代科幻风。
- 地面为灰青砖格，墙体有裂缝和符纸；房间边缘带轻微暗角。
- 所有功能性效果均须有颜色区分：敌对投射物橙红、玩家攻击青白、可交互物金黄、危险预警紫红。

## 2. 游戏流程与状态

### 标题页

- 标题：`诡计迷宫`，副标题：`无尽闯关`。
- `开始新局`：从第 1 关开始，带入永久装备和总金币，不带入局内武器/技能/医疗包。
- `继续纪录`：仅当已有最高纪录且金币 >= 500 时可用。扣除 500 金币，从“已解锁最高关卡”开始，初始生命为满血，给予基础武器桃木剑，不给予局内技能。按钮下显示消耗与可继续的关卡。
- `永久装备`：显示 8 个装备槽、碎片数量、已穿戴装备、装备等级和最终永久属性。
- `重置存档`：二次确认后清空 `localStorage` 中本游戏数据。

### 关卡状态机

`进入房间 -> 战斗中 -> 房间清空 -> 奖励阶段 -> 传送下一关`。

- 进入房间时生成地图、出生点、出口位置、波次计划和 0-2 个问号封印。
- 战斗中敌人按波次出现。尚未出场的敌人也应计入顶部“剩余鬼怪”计数。
- 敌人全部死亡后，停止生成，生成一个奖励宝箱；问号封印依然可交互。
- 奖励宝箱必须开启才能激活黑洞出口。问号封印可选择跳过，不阻塞通关。
- 玩家走进黑洞后，结算本关并进入下一关。第 10、20、30... 关为 Boss 房。
- 生命降至 0 时，进入死亡结算页；立即保存最高到达关卡和局外资源。

### 暂停

- 右上角暂停按钮；暂停时停止所有计时器、投射物、敌人与玩家移动。
- 弹层包含“继续”“返回标题”。返回标题也要保存。

## 3. 地图与生成

### 坐标与尺寸

- 逻辑格子大小为 32 px；地图尺寸随机为 16-20 格宽、13-16 格高，总可行走面积目标为 200-300 格。
- 每关从以下模板随机挑选，并使用确定性随机种子做镜像、缺口、障碍物变化：矩形庭院、L 型回廊、双室连接、十字厅、碎石祭坛。
- 墙不可穿越；至少 70% 地面互相连通。
- 玩家出生点周围半径 2 格内无障碍、无敌人；出口和宝箱预留点距出生点至少 8 格。
- 障碍物不超过可走格的 15%，不得堵死通道；敌人生成点必须在可达区域且距离玩家至少 5 格。

### 生成验收

- 连续生成 100 张地图，不得出现玩家或出口被封死的情形。
- 为移动和投射物分别实现墙体碰撞。敌方远程投射物碰墙销毁。

## 4. 玩家与数值公式

### 基础属性

| 属性 | 基础值 | 说明 |
|---|---:|---|
| 最大生命 | 300 | 开局满血 |
| 攻击 | 30 | 影响武器基础伤害 |
| 防御 | 20 | 影响伤害减免 |
| 移速 | 3 格/秒 | 逻辑速度 96 px/s |
| 攻击间隔 | 0.75 秒 | 用于保证手机操作节奏；原设定 2.5 秒过慢 |
| 暴击率 | 5% | 暴击倍率固定 1.75 倍 |

### 伤害规则

```
最终伤害 = max(1, round(攻击方攻击力 * 技能倍率 * 暴击倍率 * 100 / (100 + 防御方防御)))
```

- 未特别说明的武器技能倍率为 1.0。
- 受击后 0.25 秒无敌，避免同帧多段伤害。
- 负面状态同类效果只刷新持续时间，不无限叠层；幽灵的流血最多 3 层。
- 所有百分比加成统一按乘算前加法累计，例如两项 +10% 攻击 = 攻击 x 1.20。

### 输入

- 左下固定虚拟摇杆控制移动；右下攻击按钮，按住则朝右摇杆/最后瞄准方向自动按攻击间隔攻击。
- 右侧中部交互按钮：靠近宝箱、问号封印或黑洞时出现。
- 桌面：WASD 或方向键移动；鼠标位置决定朝向；左键或空格攻击；E 交互；Esc 暂停。
- UI 区域不应被战斗触摸事件穿透。

## 5. 敌人与关卡难度

### 敌人通用规则

- 碰撞半径：玩家 0.25 格，普通敌人 0.25 格，Boss 1.0 格。
- 普通敌人移速 1.5 格/秒；近战攻击距离 1 格、间隔 1.5 秒；远程攻击距离 8 格、间隔 3 秒。
- 近战/远程出现比例保持 80% / 20%，但每种敌人按解锁关卡加入候选池。
- 普通关分批生成，每 3 秒生成一波，单波 3-8 个，当前存活数不超过 55。
- 关卡目标敌人数不再按原案无限线性增长：

```
普通关总数 = min(18 + 关卡 x 3, 70)
Boss 关总数 = 1 个 Boss + min(22 + 关卡, 50) 个杂兵
普通敌人属性倍率 = 1 + 0.12 x (关卡 - 1)
```

- 前三关可以只生成骷髅兵、僵尸和骷髅射手；第 11 关加入幽灵和狼兵；第 31 关加入妖狐。

### 敌人表

| 敌人 | 类型 | 解锁 | 攻击/生命 | 特殊行为 |
|---|---|---:|---:|---|
| 骷髅兵 | 近战 | 1 | 30 / 50 | 直接追击 |
| 僵尸 | 近战 | 1 | 30 / 100 | 直接追击，轻微击退抗性 |
| 骷髅射手 | 远程 | 1 | 50 / 30 | 每次朝玩家方向射 2 支有小夹角的箭 |
| 幽灵 | 远程 | 11 | 70 / 70 | 发射 5 枚扇形法球；命中施加 3 秒流血，每秒损失当前生命 1%，最多 3 层 |
| 狼兵 | 近战 | 11 | 60 / 300 | 每 5 秒准备 0.5 秒后冲锋 1 秒，速度 3 格/秒；撞墙停止 |
| 妖狐 | 近战 | 31 | 100 / 100 | 命中施加 3 秒混乱，移动输入左右反转；只刷新时长，不叠加 |

### 颜色阶级

- 阶级只表达敌人强化，不另行改动表内机制。
- 根据“本关基础攻击 x 随机精英系数”决定：白 < 100，绿 < 200，蓝 < 350，紫 < 550，金 < 800，红 >= 800。
- 每生成一个敌人有 `min(0.03 + 关卡 x 0.004, 0.25)` 概率获得 1 个阶级提升，并将生命与攻击各提高 25%。
- 视觉上用描边/头顶菱形标识阶级，不能只换颜色而不可辨认。

### Boss

- 每 10 关生成 1 个 Boss，从该关已解锁的敌人类型中随机选取。
- Boss 半径为普通敌人的 4 倍（视觉面积约 16 倍），而不是原设定的 500% 面积，以避免狭小房间不可躲避。
- 基础生命和攻击为对应普通怪的 8 倍和 2.5 倍，再乘 `1 + 0.30 x (Boss序号 - 1)`；防御为 30 + 10 x (Boss序号 - 1)。
- Boss 移速 2 格/秒，继承原型技能，攻击前增加 0.4 秒可见预警。
- 每 15 秒施放“迷魂震荡”：全屏 1 秒眩晕，造成玩家最大生命 2% 的真实伤害。释放前 1.2 秒显示红色环形读条；第 1 秒后玩家可使用一次闪避取消眩晕（首版可先实现预警和眩晕）。

## 6. 武器、技能与局内物品

### 武器

- 玩家默认持有桃木剑。掉落新武器时显示“替换当前武器”确认窗；确认替换后旧武器消失。
- 所有武器基础伤害为 `玩家攻击 x 1.0`，攻击间隔沿用玩家基础攻击间隔，特殊效果见下表。

| 武器 | 权重 | 攻击距离 | 范围与效果 |
|---|---:|---:|---|
| 桃木剑 | 20 | 2 格 | 前方 90 度扇形，倍率 1.0；装备后攻击 +10% |
| 雷符 | 20 | 5 格 | 命中点半径 1.5 格爆炸，倍率 0.9；暴击率 +15% |
| 八卦盘 | 20 | 1.4 格 | 自身周围 360 度攻击，倍率 0.85；防御 +15% |
| 红葫芦 | 20 | 3 格 | 前方 3 x 1 格直线，倍率 1.0；命中附加 2 秒灼烧，每秒目标最大生命 1%，对 Boss 每秒最多 80 点 |
| 拂尘 | 20 | 2 格 | 自身前方 220 度扫击，倍率 0.9；进入房间后每 5 秒回复 5 生命，房间结束后失效 |

### 技能石

- 技能石掉落后自动停止战斗并弹出 3 张选项卡，玩家必须选 1 个后继续。
- 每张卡先按稀有度权重抽取，再从该稀有度的效果池抽取；同一次三选一不得出现重复的“属性+数值”组合。

| 稀有度 | 权重 | 数值 | 可选效果 |
|---|---:|---:|---|
| 白 | 40 | +20% | 暴击率、攻击、攻击速度、最大生命 |
| 蓝 | 30 | +30% | 暴击率、攻击、攻击速度、最大生命 |
| 紫 | 20 | +40% | 暴击率、攻击、攻击速度、最大生命 |
| 金 | 10 | +60% | 暴击率、攻击、攻击速度、最大生命 |

- 攻击速度加成降低攻击间隔：`实际间隔 = 0.75 / (1 + 攻速加成)`，下限 0.25 秒。
- 最大生命增加时，按增加后的最大生命等额增加当前生命，避免选到生命技能反而亏血。

### 医疗包

- 自动拾取；立即恢复 20% 当前最大生命，且不超过最大生命。
- 当生命已满时不自动消耗，留在地面直到本关结束。

## 7. 奖励、局外养成与存档

### 奖励宝箱

- 敌人清空后在安全位置生成。开启时使用**不放回权重抽取**，抽取 2-4 种不同奖励，避免“随机 2-6 项”与概率不闭合。
- 每类奖励最多出现一次，权重如下：金币 30、医疗包 20、随机武器 10、技能石 30、装备碎片 5、神秘宝箱碎片 5。
- 金币数量随机 10-50；医疗包为 1；装备碎片与神秘箱碎片各为 1。
- 金币、装备碎片、神秘宝箱碎片在获得瞬间写入本局可带出库存，并在死亡/返回标题时合并到永久存档。其余局内物品只对当局有效。

### 问号封印

- 每关生成 0-2 个，只在敌人清空后激活。
- 玩家靠近 1 格内，长按/站定交互 5 秒解锁；移动、受伤或离开范围均中断，进度归零。
- 使用与奖励宝箱相同的 2-4 项不放回抽取逻辑。每关最多两个。
- 显示环形进度条、剩余秒数和中断原因提示。

### 装备碎片与永久装备

| 槽位 | 碎片权重 | 完整装备效果 |
|---|---:|---|
| 头部 | 10 | 最大生命 +10%，每秒恢复 2% 最大生命；每秒恢复上限 12 |
| 左臂 | 15 | 攻击 +30 |
| 右臂 | 15 | 攻击 +5% |
| 左腿 | 15 | 防御 +5 |
| 右腿 | 15 | 防御 +5% |
| 胸部 | 15 | 最大生命 +200 |
| 背翼 | 10 | 防御 +10%，移速 +2 格/秒，负面状态时长 -30% |
| 光环 | 5 | 防御 +20%，攻击 +20%，2 格内每秒 50 伤害，5% 吸血（每秒最多回复 15） |

- 3 个同槽碎片自动合成 1 件装备。装备自动穿戴，重复完整装备转化为 1 个“装备经验”。
- 每拥有 3 个装备经验，装备等级 +1，最多 10 级。每级使来自“生命和攻击”的永久装备数值提高 10%；防御、移速、状态减免、光环范围不升级。
- 永久属性在开始一局时统一结算，局中装备页只读。

### 神秘宝箱

- 累积 3 个神秘宝箱碎片自动合成一个神秘宝箱。
- 在标题页的永久装备界面可开启；必得 200 金币和 3 个随机装备碎片。三个碎片独立抽取，允许重复。

### 存档结构

```ts
interface SaveData {
  version: 1;
  gold: number;
  highestFloor: number;
  equipmentFragments: Record<EquipmentSlot, number>;
  ownedEquipment: Record<EquipmentSlot, number>;
  equipmentExperience: number;
  equipmentLevel: number;
  mysteryFragments: number;
  mysteryChests: number;
  settings: { soundEnabled: boolean; vibrationEnabled: boolean };
}
```

- 首次启动使用默认存档。
- 每次获得局外资源、创造新纪录、死亡、返回标题时保存。
- 读取失败时回退至默认存档并在控制台输出警告，不能导致无法进入游戏。

## 8. UI、反馈与可用性

### 战斗 HUD

- 顶部左侧：关卡、当前/最大生命条、攻击、金币、剩余鬼怪数。
- 顶部中间：`种子: XXXXXXXX`，可复制或点击重新开始同一局（开发模式才显示重开）。
- 顶部右侧：暂停图标。
- 屏幕下方：左移动摇杆；右攻击摇杆/攻击键；中右交互键只在可交互时显示。
- 伤害数字最多同时显示 20 个，超过时合并或丢弃最旧数字。

### 关键反馈

- 玩家攻击：0.1 秒青白色扇形/圆形/直线闪光。
- 命中：敌人短暂白闪和受击退效果；暴击显示更大黄字。
- 受伤：屏幕边缘红闪，生命条平滑过渡。
- Boss 技能：强预警、倒计时和明显音效占位（可以用简单 WebAudio 合成音）。
- 高对比度：所有状态除颜色外要有图标或文字标签，避免只靠颜色区分。

### 性能措施

- 敌人、投射物、伤害数字、掉落物使用对象池。
- 单屏限制：普通敌人 55、玩家投射物 12、敌方投射物 80、掉落物 20。
- 超出上限时优先不生成新的视觉投射物或延迟波次，不能崩溃。

## 10. 完成验收清单

- [ ] `npm install` 后可用 `npm run dev` 启动，`npm run build` 成功。
- [ ] 在浏览器宽度 360 px 下，所有按钮可见、可触摸，文字不溢出。
- [ ] 可从标题页开始一局，使用触屏和键鼠均能移动、攻击、交互、暂停。
- [ ] 连续通关到第 10 关可触发并击败 Boss，至少出现一次宝箱和一次技能三选一。
- [ ] 关卡清空、问号封印 5 秒交互、黑洞传送的状态不会卡死。
- [ ] 金币/碎片在死亡后仍保留；局内武器/技能在新局消失；500 金币续关逻辑正确。
- [ ] 刷新页面后最高关、金币、碎片、装备都能恢复。
- [ ] 敌人、投射物和地图碰撞正常，单屏活动实体达到限制时仍稳定运行。
- [ ] 不使用外部网络资源；断网也能在本地运行。

## 11. 明确不做的内容（首版范围）

- 不做联网、排行榜、账号、充值、广告、多语言和真实美术资源加载。
- 不做复杂剧情、NPC 对话、手工关卡编辑器、云存档、多人联机。
- 不做原生 APK 打包；首版先保证移动浏览器/PWA 式网页可玩。确认网页版本后，再使用 Capacitor 打包成 Android APK。
